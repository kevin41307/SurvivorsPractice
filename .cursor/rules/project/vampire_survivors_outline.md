## 需求大綱
本產品是一款「生存 + 自動攻擊」類型的動作遊戲。玩家只需要控制角色移動、撿道具、做升級選擇，就要在怪物潮中撐到關卡時間結束。

遊戲的重點是「越玩越強」：一開始只有一把武器，打怪撿經驗後會升級；升級時可以選擇新武器或讓既有武器變強；打敗強敵會掉寶箱，寶箱能提供更多升級，並在符合條件時讓武器進化成更強版本。

一場遊玩稱為一個 **Run（一次出擊）**。Run 結束後會結算金幣與成就，玩家可以用金幣購買永久強化（MetaProgress），讓下一場更容易或有更多選擇。

### 產品目標
- **主要目標**：讓玩家在一場 Run 中活到關卡時間上限（常見 15/20/30 分鐘）。
- **次要目標**：
  - 透過解鎖系統逐步開放角色、武器、關卡與玩法選項。
  - 透過永久強化（PowerUps）讓玩家能感受到長期成長。
  - 讓玩家透過「選擇」形成不同 Build（本場武器/道具組合），每場體驗都有差異。

### 核心體驗（給第一次玩的高中生）
- **你不用一直按攻擊鍵**：多數武器會自動攻擊。
- **你最重要的事**：
  - 走位躲怪（不要被包圍）
  - 撿經驗（讓自己升級）
  - 在升級畫面做出選擇（決定這一場要走哪種路線）


## 詞彙表 (Glossary)
為了讓團隊講同一種語言，下面是本產品常用詞彙，用「一句話 + 例子」說明。

- **Run**：一次出擊的完整回合，從進關卡到死亡/結算。例：你選角色進森林玩到 30:00 結束，就是一個 Run。
- **關卡（Stage）**：地圖與規則的集合（時間上限、怪物波次、地形限制、事件）。例：圖書館是狹長走廊、森林是開闊地。
- **角色（Character）**：玩家操控的單位，有起始武器與天生特性。例：某角色開場自帶鞭子。
- **敵人（Enemy）**：會靠近玩家，碰到就造成傷害的單位。例：蝙蝠、骷髏。
- **Boss（強敵）**：較強的敵人，常會掉寶箱。例：某分鐘出現的大型怪。
- **武器（Weapon）**：用來打怪的攻擊方式，多數自動發動。例：飛刀、魔杖、聖水。
- **被動道具（Passive Item）**：不直接攻擊，而是讓你變強的加成。例：菠菜讓傷害提高。
- **升級（Level Up）**：撿到足夠經驗後觸發，遊戲會暫停讓你從 3–4 個選項選 1 個。例：拿新武器或把武器升到更高等級。
- **背包上限（6 格規則）**：升級選項最多拿到 6 把武器、6 個被動道具；但可以靠地圖上撿到的道具讓總持有超過 6。例：先用升級把被動塞滿，再去地圖撿一個額外被動。
- **寶箱（Treasure Chest）**：Boss 掉落物，打開會給金幣與升級，並可能觸發武器進化。例：10:00 後的銀寶箱常能進化武器。
- **進化（Evolution）**：把基礎武器變成更強版本，通常需要「武器滿等 + 對應被動 + 寶箱」才能完成。例：武器滿等後，開到能進化的寶箱會把它變成進化武器。
- **永久強化（MetaProgress / PowerUps）**：用金幣買的永久加成，對所有角色生效。例：永久增加傷害 5%（可買多段）。
- **極限突破（Limit Break）**：開啟後，滿裝再升級不給金幣/回血，改成讓武器繼續加數值。例：滿裝後每次升級都在強化武器的數值。
- **結算畫面（Results）**：Run 結束後的統計頁，顯示存活時間、金幣、擊殺數、各武器傷害等。例：你可以看到哪把武器打最多傷害。


## 遊戲流程 (Game Flow)
以下用「一個人第一次玩」的角度描述整體流程。

1. **進入主選單（Meta 層）**
   - 玩家可以：選角色、選關卡、開啟某些模式、購買永久強化、查看解鎖條件。

2. **開始一場 Run**
   - 玩家選擇 1 個角色 + 1 個關卡 +（可選）模式/加成。
   - 角色帶著自己的起始武器進入關卡。

3. **進行中（核心循環）**
   - 怪物從四面八方靠近你。
   - 你靠移動躲開怪物並靠武器自動攻擊清怪。
   - 打倒敵人會掉經驗寶石；撿起後增加經驗值。
   - 經驗足夠時觸發升級：
     - 遊戲暫停
     - 提供 3 或 4 個不重複的選項（武器或被動道具）
     - 玩家選 1 個
     - 遊戲恢復並給角色短暫無敵（避免剛升級就被撞死）

4. **Boss 與寶箱**
   - 某些波次會出現 Boss。
   - 打倒 Boss 可能掉寶箱。
   - 撿寶箱後會得到金幣，並讓你現有的武器/被動獲得升級；若符合進化條件，可能直接進化武器。

5. **結束條件**
   - 角色死亡：HP 變成 0，若有復活次數則復活；沒有就結束 Run。
   - 撐到時間上限：關卡視為完成；之後會出現終結機制（例如收割者類敵人）讓 Run 結束（若未開無限模式）。

6. **結算（Results）**
   - 顯示：存活時間、金幣、等級、擊殺數、武器傷害統計、這場拿到的武器/被動與等級、啟用的永久強化等。
   - 回到主選單，用金幣購買角色或永久強化，準備下一場。


## 職責和行為
（寫法：先寫定義與關係、再寫數量要求、再寫可程式化的行為步驟。）

### 遊戲本體的職責和行為
#### 定義
- 遊戲本體是整個產品的總管，負責：主選單、存檔、解鎖、啟動 Run、結算、回到主選單。

#### 關係
- 遊戲本體建立 Run，並在 Run 結束後把結果回寫到 MetaProgress（永久進度）。
- 遊戲本體管理可用的角色清單、關卡清單、解鎖清單。

#### 數量要求
- 同一時間最多只能有 1 個 Run 進行（單人前提）。
- 新手存檔至少要有 1 個可用角色、1 個可用關卡。

#### 行為（程式步驟/規則）
- **開始遊戲時**：
  - 載入存檔（MetaProgress、解鎖狀態、金幣、永久強化）。
  - 建立主選單可用清單（可選角色/可選關卡/可開啟模式）。
- **玩家按下開始 Run 時**：
  - 驗證已選：角色、關卡、模式設定。
  - 建立 Run（包含角色、關卡規則、永久強化快照）。
- **Run 結束時**：
  - 產生 Results（統計與獎勵）。
  - 把金幣/解鎖進度回寫 MetaProgress。
  - 回到主選單。


### Run 的職責和行為
#### 定義
- Run 是一次出擊的狀態容器，負責：時間、角色狀態、敵人生成、掉落物、升級流程、勝負判定與結算資料收集。

#### 關係
- Run 由遊戲本體建立；Run 內使用 1 個關卡（Stage）的規則。
- Run 內包含 1 個玩家角色（Character）、多個敵人（Enemy）、掉落物（經驗/金幣/寶箱）以及玩家的 Build（武器/被動組合）。

#### 數量要求
- 1 個 Run 必須綁定 1 個 Stage。
- 1 個 Run（單人）必須綁定 1 個玩家角色。
- 場上同時存活的敵人數量需要上限控制：當存活敵人達到 300 以上時，停止常規刷怪（Boss/事件例外）。

#### 行為（程式步驟/規則）
- **每個更新週期**：
  - 更新計時器與關卡時間（例如顯示 mm:ss）。
  - 更新敵人生成器（依波次/事件規則）。
  - 更新角色移動與碰撞（受傷/撿拾）。
  - 更新武器觸發（依武器冷卻與行為）。
- **經驗達標時觸發升級**：
  - 暫停遊戲邏輯（至少暫停敵人與武器的更新）。
  - 生成 3 或 4 個不重複選項給玩家選 1。
  - 套用選擇（新增或升級武器/被動）。
  - 給短暫無敵時間後恢復遊戲。
- **若玩家背包已滿且沒有任何可升級項目**：
  - 升級改提供「金幣」或「回血」類選項（若開啟 Limit Break 則改提供 Limit Break 強化選項）。
- **死亡/完成條件達成時**：
  - 停止 Run，產生 Results 資料並交回遊戲本體。


### 關卡的職責和行為
#### 定義
- 關卡是地圖與規則包，決定：時間上限、怪物波次、地形限制、可互動物件、地圖固定道具（Stage items）。

#### 關係
- Stage 被 Run 使用，提供刷怪與事件規則給 Run。
- Stage 可以預先放置可撿拾的武器/被動道具/遺物等。

#### 數量要求
- 每場 Run 只能有 1 個 Stage。
- Stage 必須定義 1 個時間上限（常見 15/20/30 分鐘）。
- Stage 可定義 0~多個固定道具生成點。

#### 行為（程式步驟/規則）
- **波次刷怪**：
  - 以「每分鐘一波」為主。
  - 敵人通常從畫面外生成並往玩家靠近。
  - 若場上活著敵人 ≥ 300，停止常規刷怪（Boss/事件例外）。
- **地圖互動**：
  - 生成可破壞物（例如燈），被武器打到會掉落拾取物（例如金幣、回血等）。
  - 可選：支援關卡特殊互動（壓力板、傳送門、拉桿等）。
- **完成判定**：
  - 到達時間上限後標記關卡完成，並啟動終結機制（非無限模式）。


### MetaProgress 的職責和行為
#### 定義
- MetaProgress 是跨 Run 保留的永久進度，包含：金幣、解鎖內容、永久強化（PowerUps）。

#### 關係
- 遊戲本體讀寫 MetaProgress；Run 結束後把結果回寫。
- PowerUps 會影響所有角色在 Run 中的基礎能力（例如傷害、移動速度、幸運等）與某些操作資源（例如 reroll 次數）。

#### 數量要求
- 每個存檔 1 份 MetaProgress。
- 每個 PowerUp 有自己的最大等級（例如 Might 5 級、Armor 3 級等）。

#### 行為（程式步驟/規則）
- **購買永久強化**：
  - 檢查金幣是否足夠。
  - 扣款並將該 PowerUp 等級 +1（不超過 Max）。
  - 立即影響後續 Run。
- **解鎖與購買內容**：
  - 依解鎖條件把內容標記為可用/可購買。
  - 角色多數需要花金幣購買後才可使用（可保留例外規則）。


### 角色的職責和行為
#### 定義
- 角色是玩家操控的單位，負責移動、承受傷害、持有武器與被動道具、透過升級形成 Build。

#### 關係
- 角色存在於 Run 中，並與敵人、拾取物、寶箱互動。
- 角色的能力會被被動道具與永久強化共同影響。

#### 數量要求
- 角色在 Run 開始時至少有 1 把起始武器。
- 角色透過升級選項最多取得 6 把不同武器、6 個不同被動道具。
- 角色可以透過撿地圖道具，使總持有超過 6（武器/被動皆可能）。

#### 行為（程式步驟/規則）
- **移動**：
  - 讀取輸入並更新位置（受 Move Speed 影響）。
- **受傷與死亡**：
  - 與敵人接觸時扣血。
  - HP ≤ 0 時若有 Revival 則復活並繼續，否則 Run 結束。
- **升級選擇**：
  - 套用升級結果（新增或升級武器/被動道具），並更新 Build 狀態。


### 敵人的職責和行為
#### 定義
- 敵人是朝玩家靠近並造成壓力的單位，多數以接觸方式造成傷害。

#### 關係
- 敵人由 Stage/Run 的規則生成。
- 敵人會被武器傷害；死亡後會掉落經驗等拾取物；Boss 可能掉寶箱。

#### 數量要求
- 場上同時存活敵人數量需受上限控制（≥300 時停止常規刷怪）。

#### 行為（程式步驟/規則）
- **生成**：
  - 依波次在畫面外生成，並開始朝玩家移動。
- **攻擊**：
  - 與玩家接觸時造成傷害（受玩家減傷/護甲等影響）。
- **死亡掉落**：
  - 死亡後依規則掉落經驗寶石；Boss 依設定機率掉落寶箱。
- **刷怪規則**：
  - 若玩家移動很遠，允許某些敵人被系統移除以維持效能。


### 武器的職責和行為
#### 定義
- 武器是角色的攻擊方式，通常自動觸發，具備冷卻與攻擊規則。

#### 關係
- 武器屬於角色；武器攻擊敵人並造成傷害或狀態效果。
- 武器可升級到滿等，並可能在寶箱中進化成更強版本。

#### 數量要求
- 角色透過升級選項最多擁有 6 把不同武器。
- 多數武器同一場 Run 只能取得 1 次（少數特殊一次性道具例外）。

#### 行為（程式步驟/規則）
- **取得**：
  - 起始武器由角色自帶；其他武器主要由升級選項取得，或由地圖道具撿到。
- **升級**：
  - 玩家選到已擁有武器時，武器等級 +1。
  - 升級後重設冷卻並立即觸發一次（用於即時戰力提升）。
  - 武器滿等後不再出現在升級選項中。
- **進化**：
  - 當「武器滿等 + 對應被動已取得 + 撿到可進化寶箱」時觸發。
  - 進化時一般規則為：移除基礎武器、加入進化武器，進化武器從冷卻 0 開始。


### Build 的職責和行為
#### 定義
- Build 是玩家在一場 Run 裡湊出的組合：武器清單 + 被動道具清單 + 等級/進化狀態。

#### 關係
- Build 由角色持有並在 Results 中被統計與顯示。
- Build 影響玩家清怪方式與能否進化武器。

#### 數量要求
- 來自升級選項的核心上限：
  - 武器最多 6 把
  - 被動道具最多 6 個
- 可透過撿地圖道具使總持有超過 6（需要系統支援顯示與升級）。

#### 行為（程式步驟/規則）
- **每次升級選擇後更新 Build**：
  - 新增項目或提升項目等級。
- **Build 必須能被查詢**：
  - 是否已滿裝（武器 6 + 被動 6）。
  - 是否存在可升級項目。
  - 是否存在可進化武器（武器滿等 + 對應被動到手）。


### 寶箱的職責和行為
#### 定義
- 寶箱是 Boss 掉落的獎勵容器，提供金幣與多次升級，並可能觸發進化。

#### 關係
- 寶箱由 Boss 掉落，玩家撿到後開啟並套用獎勵到自己的武器/被動。

#### 數量要求
- 一個寶箱通常提供 1、3 或 5 次獎勵格（視品質與規則）。
- 若玩家所有道具都滿等且不可進化，寶箱道具獎勵應替換為純金幣袋（只給錢）。

#### 行為（程式步驟/規則）
- **掉落**：
  - 只從 Boss 掉落，不從一般可破壞物掉落。
  - 不會自動吸到玩家，玩家需走過去撿。
- **開啟**：
  - 給予金幣（可受金幣倍率影響）。
  - 依寶箱獎勵格數，對玩家已擁有的武器/被動進行升級。
  - 若符合進化條件且此寶箱允許進化，優先把「進化」作為獎勵結算（通常是第一格）。


## 根據行為變動性，值得用到策略模式、責任鏈模式、狀態模式、指令模式、裝飾者模式的地方
本章節的目的：當你預期「同一種概念」會一直加新種類（新敵人、新武器、新規則、新 UI 操作）時，優先用設計模式把變動點隔離，避免滿滿 if-else。

### 策略模式（Strategy）：同一流程，不同演算法
適用特徵：流程固定（更新/結算順序不變），但「某一步怎麼做」會常常換。

- **敵人移動（MoveBehavior）**
  - **Open for**：追蹤玩家、直線固定方向、飄移/波浪、衝刺掃過、繞行、保持距離。
  - **Close for**：敵人更新生命週期（Update/FixedUpdate）、位置/速度資料結構、碰撞與受擊入口。
- **敵人死亡掉落（DropBehavior）**
  - **Open for**：掉經驗、掉寶箱、掉特殊道具、依條件掉落（例如只在 Boss/事件中掉）。
  - **Close for**：死亡事件觸發、掉落生成位置/物件池、結算統計管線。
- **抗性與受控反應（CrowdControlResponse / Resistances）**
  - **Open for**：抗凍結、抗擊退、抗秒殺、抗減益等規則；不同敵人吃效果的比例/上限。
  - **Close for**：武器命中事件、狀態效果套用的共用流程。
- **武器觸發方式（WeaponFireBehavior）**
  - **Open for**：打最近目標、固定方向射擊、範圍持續、放置地板傷害、召喚物跟隨等。
  - **Close for**：武器冷卻計時、傷害結算入口、武器等級提升流程。

### 責任鏈模式（Chain of Responsibility）：很多條件判斷，想要可插拔
適用特徵：一連串檢查/處理步驟，會因為新增規則而插入、移除或改順序。

- **升級選項生成與過濾（Level-up Option Pipeline）**
  - 範例責任節點：
    - 依解鎖狀態建立候選池 → 套用權重/稀有度 → 移除已滿等 → 移除背包滿時不可新增 → 套用禁用/封印（Seal/Banish）→ 產生 3–4 個不重複選項
  - **Open for**：新增過濾規則（例如某模式禁止某類道具、某角色偏好某類道具）。
  - **Close for**：升級 UI 流程（暫停→顯示→套用→恢復）與「最後一定產出可用選項」的保底機制。
- **傷害結算管線（Damage Resolution Pipeline）**
  - 範例責任節點：無敵判斷 → 護甲/減傷 → 抗性 → 吸血/反傷 → 最終扣血 → 觸發受傷事件
  - **Open for**：新增防禦規則或特殊效果節點。
  - **Close for**：唯一的傷害入口與事件順序，避免各處自行扣血。

### 狀態模式（State）：同一物件，不同狀態下行為不同
適用特徵：物件會在少數幾個狀態間切換，且狀態會改變「能不能做事」或「怎麼做事」。

- **角色生命狀態（Alive / Invulnerable / Dead）**
  - **Open for**：新增狀態（例如短暫無敵、暈眩、沉默、被詛咒等）。
  - **Close for**：角色更新主迴圈與「死亡→復活→結束 Run」的關鍵轉移流程。
- **Run 狀態（Running / LevelUpPaused / Results）**
  - **Open for**：新增暫停原因（升級、開寶箱、過場、商人交易）。
  - **Close for**：Run 狀態轉移規則（避免同時又暫停又更新世界）。
- **敵人狀態（Normal / Frozen / Stunned / Despawning）**
  - **Open for**：新增受控狀態與對移動/受擊的影響。
  - **Close for**：敵人共同生命週期與清理規則。

### 指令模式（Command）：把「玩家操作」變成可記錄、可重播、可撤銷（或可同步）
適用特徵：你想把操作抽成「一個指令」，用於重播、教學、測試、連線同步或 Undo。

- **升級選擇（SelectLevelUpOptionCommand）**
  - **Open for**：新增可選行為（選項類型變多、一次性特殊選項）。
  - **Close for**：套用升級結果的核心服務（新增/升級/檢查上限/更新 UI）。
- **購買永久強化（BuyPowerUpCommand）/ 購買角色（BuyCharacterCommand）**
  - **Open for**：新增商店行為、不同貨幣、折扣活動。
  - **Close for**：金幣扣款與存檔寫入的單一入口（避免金幣被多處改動）。
- **關卡前設定（ToggleModifierCommand：Hyper/Hurry/Inverse/Endless/LimitBreak…）**
  - **Open for**：新增模式開關與條件。
  - **Close for**：設定存取與套用流程。

### 裝飾者模式（Decorator）：把能力加成「一層一層疊上去」
適用特徵：很多加成可以同時存在，且想要自由疊加、自由拿掉，不想寫爆炸多的組合分支。

- **角色數值加成（Stats Modifiers）**
  - 來源：被動道具、永久強化、關卡修正、模式修正。
  - **Open for**：新增新的加成種類（傷害、範圍、冷卻、移速、幸運…）與疊加規則。
  - **Close for**：最終數值計算入口（統一從「基礎值 → 套用一串修飾器 → 得到最終值」）。
- **武器行為加成（Weapon Modifiers）**
  - 例：同一把武器因為道具/模式而多射 1 發、冷卻更短、範圍更大。
  - **Open for**：新增武器修飾器。
  - **Close for**：武器原始行為與觸發流程。


## 值得用到單體模式、工廠方法、抽象工廠的地方
本章節的目的：把「全域唯一的服務」與「大量產生的物件」分清楚，並讓新增內容時不需要改到既有流程。

### 單體模式（Singleton）：全域唯一且跨處共用的服務
適用特徵：在一個執行中的遊戲流程中，理論上應該只存在 1 份，且很多地方都需要讀寫它。

> 注意：單體好用但容易失控。若你的專案已使用 DI（依賴注入），多數情況建議「在容器中註冊成單例生命週期」取代硬 Singleton。

- **存檔/永久進度服務（MetaProgressRepository / SaveService）**
  - 用途：讀寫金幣、已解鎖內容、永久強化。
  - 好處：避免各系統各自寫檔造成衝突。
- **全域時間/時鐘（GameClock / TimeProvider）**
  - 用途：提供 Run 計時、冷卻計算的統一時間來源（方便測試/回放）。
- **事件匯流排/訊息系統（EventBus）**
  - 用途：角色死亡、升級、掉落、寶箱開啟、關卡完成等事件集中派送。
- **隨機數來源（RandomProvider / RNG）**
  - 用途：升級選項抽取、掉落機率、寶箱品質；統一來源方便重播與測試。

### 工廠方法（Factory Method）：同一類產品，但要依資料/條件產生不同子類
適用特徵：你要「產生一個東西」，但它的具體型別/行為會隨著 ID、關卡、模式或設定不同。

- **敵人產生（EnemyFactory.Create(enemyId, context)）**
  - 會變動的部分：MoveBehavior、Resistances、DropBehavior、Boss/DespawnPolicy。
  - 固定不想改的部分：Run 的刷怪流程、上限 300 的規則、生成點選擇框架。
- **武器產生（WeaponFactory.Create(weaponId, owner, context)）**
  - 會變動的部分：攻擊型態、冷卻、投射物、命中效果、進化後替換。
  - 固定不想改的部分：武器冷卻更新入口、傷害結算入口、升級套用流程。
- **掉落產生（DropFactory.Create(dropTableId / enemyId, context)）**
  - 會變動的部分：經驗寶石、金幣、寶箱、特殊道具。
  - 固定不想改的部分：死亡事件 → 生成掉落 → 撿拾處理 → 統計記錄。

### 抽象工廠（Abstract Factory）：同一組「互相搭配的一整套物件」，需要成套切換
適用特徵：你不只要產生單一物件，而是要產生「一整組互相相容」的產品族群（同一套規則與風格）。

- **Stage 內容工廠（IStageContentFactory）**
  - 產物族群：
    - 波次/事件規則（Wave/Event）
    - 敵人配置（哪些 enemyId 會出現）
    - 地圖固定道具（stage items）
    - Boss 與寶箱配置
  - 何時用：新增關卡或切換模式（例如一般/倒轉/加速）時，希望「換一個工廠就換一整套內容」，不改 Run。
- **模式/規則工廠（IModifierFactory / IRunRulesFactory）**
  - 產物族群：
    - 角色與敵人的數值修正器（Decorator/Modifier）
    - 升級選項管線的節點集合（Chain pipeline）
    - 掉落規則調整（例如某模式改變金幣倍率或寶箱行為）
  - 何時用：你希望把「Hyper/Hurry/Inverse/Endless/LimitBreak」等當作一組可組合的規則包。
- **平台/輸入/呈現工廠（IPresentationFactory，可選）**
  - 產物族群：UI 呈現、輸入對應、音效/震動等平台差異。
  - 何時用：同一套 Domain 不動，但不同平台要換一整套介面或輸入層。
